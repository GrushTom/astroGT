# 友链管理

本目录用于存放友链的JSON文件，系统会自动加载并显示这些友链。

## JSON文件格式

每个友链使用一个单独的JSON文件，文件格式如下：

```json
{
  "name": "网站名称",
  "avatar": "图标URL",
  "description": "描述信息",
  "url": "链接地址"
}
```

### 字段说明

- `name`：网站名称（必填）
- `avatar`：网站图标URL（必填）
- `description`：网站描述信息（必填）
- `url`：网站链接地址（必填）

## 使用方法

1. 在本目录下创建一个新的JSON文件，文件名可以任意命名，例如 `example.json`
2. 按照上述格式填写友链信息
3. 保存文件后，系统会自动加载并显示该友链
4. 友链会按照名称A-Z排序显示

## 示例

```json
{
  "name": "示例网站",
  "avatar": "https://example.com/favicon.ico",
  "description": "这是一个示例网站",
  "url": "https://example.com"
}
```

## 注意事项

- 确保JSON文件格式正确，否则会被跳过并在控制台显示错误信息
- 所有字段都是必填的，缺少任何字段都会导致该友链被跳过
- 友链会自动与配置文件中的友链合并显示
- 系统会为JSON友链添加默认值：tags为空数组，weight为0，enabled为true

## 错误处理

如果JSON文件格式错误或缺少必填字段，系统会在控制台显示警告信息，但不会影响其他友链的加载和显示。
